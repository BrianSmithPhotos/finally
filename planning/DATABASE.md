# Database Layer — Contract for Downstream Agents

The persistence layer is a thin repository over SQLite (stdlib `sqlite3`, no ORM).
It owns connection management, lazy schema init + seeding, and all data access.
Consumed by the Backend API Engineer and LLM Engineer.

## Imports

```python
from app.db import (
    Database,            # repository class
    get_database,        # process-wide singleton accessor
    reset_database,      # drop/close the singleton (tests)
    DEFAULT_USER_ID,     # "default"
    DEFAULT_CASH_BALANCE,# 10000.0
    DEFAULT_TICKERS,     # ["AAPL", "GOOGL", ...] (10)
)
```

## Getting an instance

```python
db = get_database()        # app code: lazy-inits + seeds db/finally.db on first call
```

- DB file defaults to the project-root `db/finally.db` (the volume mount).
  Override via the `FINALLY_DB_PATH` env var or `get_database(path)` /
  `Database(path)` on first call.
- `get_database()` is a process-wide singleton; `db_path` honored only on the
  first call. Construct `Database(tmp_path)` directly in tests.
- Initialization is automatic and idempotent (creates schema if missing, seeds
  default user + watchlist if absent). Call `db.initialize()` explicitly if you
  construct with `auto_init=False`.

## Conventions

- **Return shapes:** every read/write returns plain `dict`s (or `list[dict]`),
  ready for JSON. Column names match the schema exactly.
- **IDs:** UUID4 strings, generated by the layer. **Timestamps:** ISO-8601 UTC
  strings, generated by the layer unless you pass an explicit `*_at` arg.
- **Tickers** are normalized to uppercase on write and lookup.
- **Money / quantities** are `float` (REAL); fractional shares supported.
- **`user_id`** defaults to `DEFAULT_USER_ID` on every method (single-user now).
- This layer does **not** do trade/portfolio math (weighted avg cost, P&L,
  cash validation). It persists what you give it — that logic lives in the API layer.

## Public API (`Database` methods)

### Users / cash
```python
get_user(user_id=DEFAULT_USER_ID) -> dict | None
    # {id, cash_balance, created_at}
get_cash_balance(user_id=DEFAULT_USER_ID) -> float          # raises KeyError if no user
set_cash_balance(cash_balance: float, user_id=...) -> float # absolute set; returns new
adjust_cash_balance(delta: float, user_id=...) -> float     # += delta; returns new
```

### Watchlist
```python
list_watchlist(user_id=...) -> list[dict]   # {id, user_id, ticker, added_at}, oldest first
list_watchlist_tickers(user_id=...) -> list[str]
add_watchlist_ticker(ticker: str, user_id=...) -> dict      # idempotent; returns row
remove_watchlist_ticker(ticker: str, user_id=...) -> bool   # True if removed
```

### Positions
```python
list_positions(user_id=...) -> list[dict]   # {id,user_id,ticker,quantity,avg_cost,updated_at}
get_position(ticker: str, user_id=...) -> dict | None
upsert_position(ticker, quantity, avg_cost, user_id=...) -> dict  # absolute values
delete_position(ticker: str, user_id=...) -> bool           # e.g. on full sell
```

### Trades (append-only)
```python
insert_trade(ticker, side, quantity, price, user_id=..., executed_at=None) -> dict
    # side must be "buy" | "sell" (ValueError / CHECK otherwise)
list_trades(user_id=..., ticker=None, limit=None) -> list[dict]   # newest first
```

### Portfolio snapshots
```python
insert_snapshot(total_value, user_id=..., recorded_at=None) -> dict
list_snapshots(user_id=..., limit=None) -> list[dict]   # chronological (oldest first);
    # with limit: the most-recent `limit`, still oldest-first
```

### Chat messages
```python
insert_chat_message(role, content, actions=None, user_id=..., created_at=None) -> dict
    # role must be "user" | "assistant"; `actions` is a JSON STRING you encode yourself
list_chat_messages(user_id=..., limit=None) -> list[dict]   # chronological (oldest first);
    # with limit: most-recent `limit`, oldest-first (ideal for LLM context window)
```

## Usage example

```python
from app.db import get_database

db = get_database()

# Execute the persistence side of a buy (validation/math done in API layer):
db.adjust_cash_balance(-1900.0)
db.upsert_position("AAPL", quantity=10, avg_cost=190.0)
db.insert_trade("AAPL", "buy", quantity=10, price=190.0)
db.insert_snapshot(total_value=10000.0)

# Chat history for LLM context (last 20, oldest-first):
history = db.list_chat_messages(limit=20)
```

## Schema

Defined in `backend/db/schema.sql` (six tables per PLAN.md Section 7:
`users_profile`, `watchlist`, `positions`, `trades`, `portfolio_snapshots`,
`chat_messages`). `users_profile.id`, `watchlist`/`positions` UNIQUE(user_id,
ticker), CHECK constraints on `trades.side` and `chat_messages.role`. WAL +
`foreign_keys=ON` are enabled per connection.
